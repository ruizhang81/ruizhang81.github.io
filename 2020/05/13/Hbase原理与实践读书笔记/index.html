HBase原理与实践读书笔记

概述

google三论文

1、GFS：Google File System （数据分布式存储）
2、MapReduce：Simplefied Data Processing on Large Clusters 数据的分析计算
3、BigTale：A Distributed Storage System for Structured Data 高效读写

一、HBase是在HDFS之上的分布式KV数据库系统

二、HBase数据模型:

三、HBase体系结构

1、zk保证高可用
2、hdfs保证存储
3、HBase负责读写功能

  • 写缓存 memstore (region下管理多个store 每个store有1个memstore和多个HFile)
  • 读缓存 blockCache (直接管理block)
  • 分布式 WAL(Hlog)

四、HBase系统特征

  • 优点:
    1、容量大
    2、良好的扩展性 可以简单堆机器来实现扩容
    3、稀疏性 不浪费空间
    4、高性能，随机读取及小范围读取 性能强劲
    5、数据多版本存储
    6、支持多版本数据过期删除
    7、Hadoop原生支持，可针对HBase的优化
  • 缺点
    1、不支持聚合运算
    2、技术角度不支持二级索引，业务角度支持
    3、只支持单行事务

数据结构和算法

程序=算法+数据结构

1、列簇（store）是LSM树

类似B+树 LSM树对写入更加友好，因为他会将写入操作处理成一次顺序写
内存和磁盘上都是有序的，key排序规则: rowkey+columnFamily+column+timestamp+type
内存部分（跳跃表），

2、LSM内存部分:跳跃表，增删查 O(logN) 一旦超过阈值，将会flush到磁盘上

因为不断的写入，所以会分裂，用两个memstore来保证写性能稳定

3、LSM磁盘部分:布隆过滤器

写入的时候都是顺序写，为了照顾查询的需求，所以会合并（异步）
compact分为minor和major

4、布隆过滤器

解决了如何在磁盘上查看一个元素是否在集合里

1、把集合里所有的元素hash若干下，然后组合成一个01数组K
2、将需要查询的元素也hash若干下，得到一个01数组K1，如果K包含K1，则集合里可能包含该元素，反之必不包含该元素
